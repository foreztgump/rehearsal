# v1.2 R3 — Plan B + C: Hardware-Aware Advice + Docs/Verify — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Finish R3 by making STT placement honest about real VRAM and advising an engine per hardware (Plan B), then reframing the docs around the 3-mode engine model and consolidating every R3 operator gate into one verification doc (Plan C). No change to the per-turn hot path; `streaming` stays byte-identical.

**Architecture:** Plan B is **advise-only**: `agent/placement.py` reads `VRAM_TOTAL_MB` from env so its GPU-fit ceiling is correct on a non-16GB card, and `scripts/gpu-doctor.sh` prints a recommended `STT_ENGINE`/`STT_BUFFERED_DEVICE`/`VRAM_TOTAL_MB` snippet per detected VRAM (named thresholds, never writes `.env`, always exits 0). The Parakeet onnxruntime EP is owned server-side by `STT_BUFFERED_DEVICE` — there is no agent-side device resolver and no `PARAKEET_GPU_MB` constant (both deliberately dropped; see `.planning/v1.2-R3-DESIGN.md` → "Plan B/C build refinements"). Plan C is docs + a new operator-gate doc; the dormant `finalize()` candidates stay default-OFF.

**Tech Stack:** Python (uv) `agent/placement.py` pure self-check; Bash `scripts/gpu-doctor.sh` + `scripts/test_gpu_doctor.sh` (PATH-shim harness, no real GPU); Markdown docs.

## Global Constraints

- **B and C are ONE combined plan** (this doc). Both are small and share the docs/verify deliverable.
- `placement.py` stays **pure**: no `nvidia-smi`, no live probe, **never raises** (it is a hot-path session decision, not a startup precondition). A bad/non-numeric `VRAM_TOTAL_MB` falls back to the default, never throws.
- `gpu-doctor.sh` keeps its **advise-only** posture: **never writes `.env`, always `exit 0`**. New thresholds are **named `readonly`** vars (no inline magic numbers), mirroring `VRAM_FLOOR_MB`/`CUDA_FLOOR`.

- Tests are **stdlib `assert`-based `_self_check()` run via `python3 <file>`** (placement) and the existing **PATH-shim bash harness** (gpu-doctor) — **NOT pytest**. AAA structure.
- Code rules: ≤40 lines/function, ≤3 params, ≤3 nesting, error handling on every boundary, named constants only.
- Local-first unchanged: no cloud endpoints; this plan adds no runtime dependencies.
- **Branch first:** `master` is the integration branch — create `v1.2-r3-bc` before Task 1 (`git switch -c v1.2-r3-bc`).
- Conventional Commits: `feat(r3): …` for Plan B, `docs(r3): …` for Plan C.

def _vram_total_mb(env: Mapping[str, str]) -> int:
    """Env-derived total VRAM (MB). Non-numeric/missing → the 16384 default. Never raises."""
    raw = env.get("VRAM_TOTAL_MB", "").strip()
    if not raw.isdigit():
        return VRAM_TOTAL_MB
    return int(raw)


def _gpu_fits(env: Mapping[str, str]) -> bool:
    """Worst-case-LLM headroom math (STT-06) against the ENV-derived VRAM total: does
    the HEAVIEST LLM + Kokoro + GPU-STT fit under (total − 1GB headroom)? Keying off
    ``max(LLM_PEAK_MB)`` means the decision is the SAME for fast and better, so a
    mid-session swap can never strand placement.
    """
    ceiling = _vram_total_mb(env) - VRAM_HEADROOM_MB
    worst_llm = max(LLM_PEAK_MB.values())
    return worst_llm + KOKORO_MB + STT_GPU_MB <= ceiling
```

# recommend_stt_profile <vram_mb> — echo the advised STT_ENGINE/device/total env lines
# for the detected VRAM. Pure: a number in, lines out; empty/non-numeric → conservative
# buffered/CPU. Advise-only — the operator copies these into .env; we never write it.
recommend_stt_profile() {
  local vram="$1"
  case "${vram}" in
    ''|*[!0-9]*)
      printf '  STT_ENGINE=buffered          # no/unknown VRAM -> accuracy-mode, CPU Parakeet\n'
      printf '  STT_BUFFERED_DEVICE=cpu\n'
      return ;;
  esac
  if [ "${vram}" -ge "${VRAM_FLOOR_MB}" ]; then
    printf '  STT_ENGINE=hybrid            # >=16GB -> Nemotron partials + Parakeet final\n'
    printf '  STT_BUFFERED_DEVICE=gpu\n'
  elif [ "${vram}" -ge "${VRAM_HYBRID_MB}" ]; then
    printf '  STT_ENGINE=hybrid            # 12-16GB -> hybrid, Parakeet on CPU (frees VRAM)\n'
    printf '  STT_BUFFERED_DEVICE=cpu\n'
  else
    printf '  STT_ENGINE=buffered          # <12GB -> accuracy-mode, CPU Parakeet\n'
    printf '  STT_BUFFERED_DEVICE=cpu\n'
  fi
  printf '  VRAM_TOTAL_MB=%s\n' "${vram}"
}
```
