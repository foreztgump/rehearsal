# Phase 12 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** the OPTIONAL, default-OFF 3D avatar (met4citizen/TalkingHead) added to the existing
> voice UI in `web/app/`. This phase is STRICTLY FRONTEND-ONLY behind a hard isolation gate
> (`git diff -- agent/ stt/ tts/ docker-compose.yml` MUST be empty). NO new design system,
> NO component library, NO CSS framework. This contract MATCHES the existing inline-style
> GitHub-dark theme already in `web/app/` (VoiceRoom, AgentStatePill, PersonaPanel, ModelPanel).

## Spacing Scale

The existing code expresses spacing in `rem` (1rem = 16px). Declared values map to a 4px base.

| Token | Value | rem | Usage |
|-------|-------|-----|-------|
| xs | 4px | 0.25rem | Inline gaps (label→control), pill vertical padding |
| sm | 8px | 0.5rem | Border radius, compact control padding, toggle segment gap |
| md | 16px | 1rem | Default element/panel padding, panel-row gap, marginTop after pill |
| lg | 24px | 1.5rem | Avatar stage internal breathing room / button horizontal padding (1.5rem) |

Exceptions (carried verbatim from existing components — do NOT "fix" to the pure 4-grid):
- `0.4rem` (6.4px) vertical input padding — matches PersonaPanel/ModelPanel `inputStyle`.
- `0.6rem` (9.6px) panel `gap` — matches existing `panelStyle`.
- `0.75rem` (12px) button vertical padding + pill horizontal padding — existing convention.
- `0.35rem` radius on inputs/buttons; `0.5rem` radius on panels; `999px` radius on the state pill.

**New avatar surfaces MUST reuse these exact values** so the avatar stage and toggle are visually
indistinguishable in convention from the existing panel row.

## Typography

System font stack, two weights only (400 regular / 600 semibold) — matches every existing component.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display (page H1 "Adept") | ~32px (browser default h1) | 600 | 1.2 |
| Heading (panel title `<strong>`) | 16px (1rem) | 600 | 1.2 |
| Body / control label | 14.4px (0.9rem) | 600 label / 400 input value | 1.5 |
| Caption (status, muted hints, pill, fallback copy) | 14.4px (0.9rem) | 600 | 1.4 |

Rules:
- Avatar toggle label, loading text, and fallback/error copy use **0.9rem** (the panel body size).
- Control labels are **weight 600**; values/help text are **weight 400** (existing split).
- Do NOT introduce a 5th size or a 3rd weight. No italic, no uppercase except the pill's existing
  `textTransform: capitalize`.

## Color

60/30/10 split, derived from the existing theme. Avatar surfaces inherit this split unchanged.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0b0f14` (page) / `#0d1117` (panel/stage surface) | App background; the avatar stage canvas backdrop + its host panel |
| Secondary (30%) | `#161b22` surface, `#30363d` borders, `#c9d1d9`/`#8b949e` text | Toggle track, control surfaces, panel borders, body + muted text |
| Accent (10%) | `#3fb950` (green) | See reserved-for list below |
| Destructive / error | `#f85149` (red) | Avatar load/WebGL-degrade error message text only |

Accent reserved for (explicit — never "all interactive elements"):
- The active/selected segment of the **Voice only / Avatar** toggle (the ON state indicator).
- Reuse of the existing primary-button green is ONLY for already-existing buttons; the avatar
  toggle is a segmented control, not a green CTA button.

Secondary semantic colors (already in the theme — reuse, do NOT add new hues):
- `#58a6ff` blue = "speaking" agent state (drives lip-sync gating; the avatar's speaking cue is
  the existing pill, NOT a new colored chrome on the stage).
- `#d29922` amber = transient/"thinking" + "loading avatar…" status text.
- `#3fb950` green = "listening" agent state + applied/success status.
- `#f85149` red = degrade/error copy.

Avatar-specific color rules:
- The avatar stage adds **no new background color** — its host surface is `#0d1117` (panel bg),
  bordered `1px solid #30363d`, radius `0.5rem`, exactly like a panel.
- The avatar is the agent's "face"; do NOT add colored glows/rings around it for agent state —
  state remains communicated by the existing `AgentStatePill`. (Eye-contact/mood in 12-02 are
  in-scene avatar behaviors, not UI chrome colors.)

## Copywriting Contract

All copy is sentence case, terse, no emoji (matches existing "Start talking" / "applying…" tone).

| Element | Copy |
|---------|------|
| Primary control (the toggle) | Two segments labeled **`Voice only`** and **`Avatar`**. Group is unlabeled inline (sits next to the state pill); default selected = `Voice only`. |
| Loading state | `Loading avatar…` (amber `#d29922`, 0.9rem) — shown while the dynamic import + WebGL init + GLB load run, in the stage host. |
| Empty / not-yet-on state | No avatar chrome renders when toggle = Voice only (the stage is unmounted entirely — AVTR-01). There is no empty-canvas placeholder; absence IS the empty state. |
| WebGL-unavailable / degrade | `3D avatar unavailable on this device — use Voice only.` (red `#f85149`, 0.9rem, inside the stage host; does NOT throw, no retry loop — AVTR-08). |
| Generic load error | `Couldn't load the avatar — staying in Voice only.` (red `#f85149`, 0.9rem) — on dynamic-import / GLB fetch failure. |
| Destructive confirmation | None. Toggling back to `Voice only` is non-destructive and instant (unmounts canvas + tears down AudioWorklet); no confirmation dialog. There are NO destructive actions in this phase. |

Microcopy rules:
- Use `Avatar` / `Voice only` exactly (matches REQUIREMENTS AVTR-01 wording). Not "3D", not "Face".
- Degrade copy always names the escape hatch ("use Voice only") because the toggle IS the
  documented graceful-degradation path (AVTR-08).
- Loading uses an ellipsis-character style consistent with existing `applying…`/`applied`.

## Component & Layout Contract

This section is the avatar-specific interaction/visual contract the planner + executor consume.
It is additive to `VoiceRoom.tsx`; it does NOT reorder or restyle existing panels.

### 1. Voice only / Avatar toggle (AVTR-01)
- **Type:** a 2-segment inline segmented control (two `<button>`s in a bordered pill container),
  NOT a checkbox, NOT a green CTA. Default selected segment = **`Voice only`** (`avatarOn=false`).
- **Placement:** inside `<LiveKitRoom>`, on the same row as `<AgentStatePill>`, to its right,
  with `gap: 0.5rem`. It sits ABOVE the existing panel row (which keeps `marginTop: 1rem`).
- **Visual:**
  - Container: `display:inline-flex`, `border:1px solid #30363d`, `borderRadius:999px`,
    `background:#161b22`, `padding:0.25rem`, `gap:0.25rem`.
  - Selected segment: `background:#3fb950`, `color:#0b0f14`, `fontWeight:600`, `borderRadius:999px`,
    `padding:0.25rem 0.75rem`, `border:none`.
  - Unselected segment: transparent background, `color:#8b949e`, same padding/weight, `border:none`,
    `cursor:pointer`.
  - Font: `0.9rem`. Matches the pill's metrics so the two controls read as a set.
- **Behavior:** clicking `Avatar` sets `avatarOn=true` → mounts the dynamic-imported `AvatarStage`.
  Clicking `Voice only` sets `avatarOn=false` → unmounts `AvatarStage` (full teardown). No latency
  cost when OFF; the avatar module is absent from the voice-only bundle.

### 2. Avatar stage (canvas host) — AVTR-05 framing
- **Placement:** inside `<LiveKitRoom>`, rendered between the toggle row and the panel row, i.e.
  `{avatarOn && <AvatarStage/>}` mounts ABOVE the `display:flex` panel row. The panel row is
  unchanged and remains horizontally scrollable/wrapping beneath it.
- **Framing/size:** the host is a fixed-aspect bordered surface sized for an interview-appropriate
  upper/head crop (`cameraView: "upper"` default, `"head"` alternate — AVTR-05). Recommended host
  box: `width: 20rem` (matches `panelStyle` width so it aligns with the panel column grid),
  `aspectRatio: 3 / 4` (portrait — appropriate for an upper-body/head interview crop), centered.
  The TalkingHead canvas fills the host.
- **Visual chrome:** `background:#0d1117`, `border:1px solid #30363d`, `borderRadius:0.5rem`,
  `overflow:hidden`. NO title bar, NO extra labels — the avatar is its own content. `marginBottom:1rem`
  so it sits cleanly above the panel row.
- **Coexistence:** the avatar is ADDITIVE. It must not push, hide, or restyle PersonaPanel,
  InterviewPanel, ModelPanel, KbPanel, or Transcript. On narrow viewports the existing flex row
  wraps as before; the avatar stage stacks above it.

### 3. Loading / degraded / error states (AVTR-08)
- The stage host always renders; its inner content is one of:
  - **loading:** centered `Loading avatar…` (amber) until WebGL + GLB are ready.
  - **ready:** the live TalkingHead canvas.
  - **degraded/error:** centered red fallback copy (see Copywriting). No spinner library — text only.
- The fallback is rendered INSIDE the same bordered host so layout doesn't shift between states.
- Degrade NEVER throws and NEVER auto-retries; the user's escape hatch is flipping the toggle back
  to `Voice only`.

### 4. Agent-state coupling (12-02, contract only)
- The avatar's speaking/listening eye-contact + mood are driven off `useVoiceAssistant().state`
  (the SAME source as `AgentStatePill`) — these are in-scene behaviors, not new UI chrome.
- The existing `AgentStatePill` REMAINS visible in Avatar mode (it is the canonical state readout).
  Do NOT duplicate state into colored borders/badges on the stage.
