# Plan 03-01: Persona config → frozen-prefix system prompt + voice id (default trainer unchanged)

## User Story

**As a** learner who loads the app, **I want to** still get the exact same default Cybersecurity
Trainer (greeting, gentle correction, `af_bella` voice), now driven by a structured `Persona`
config that renders a **byte-stable** system prompt, **so that** the persona is editable next
plan and Phase 4's KB cache has a frozen prefix to depend on — with zero behavior change today.

## Context

This is the **agent-side, mostly-pure** slice. It lifts the static `PERSONA_INSTRUCTIONS`
literal (`agent/main.py:62-75`) into a new pure module `agent/persona.py`: a `Persona`
dataclass, three enum→fixed-string knob tables (difficulty/verbosity/correction), a curated
Kokoro voice-id list, `render_persona(p) -> str` (frozen-prefix assembly with the empty Phase-4
KB seam), `DEFAULT_PERSONA` reproducing today's trainer, and a `_self_check()` byte-stability
test. Then `agent/main.py` is refactored to render the default, hold a **named `agent` ref**
(so 03-02's RPC handler can reach it), and source the TTS voice from the persona.

This is **MODIFY against a live analog**, not greenfield. The template for the pure module is
`agent/metrics.py` (frozen constants + `@dataclass` + `_self_check()` under
`if __name__ == "__main__":`). Mirror its shape exactly (`03-PATTERNS.md` Pattern A).

**The #1 constraint is byte-stability of the rendered persona prefix** (`03-RESEARCH.md` §3,
Pitfall 7): knobs map to **fixed-string lookups, never interpolated numbers**; the prompt is
assembled by joining **frozen constants in a fixed tuple order** with a fixed separator — no
f-strings on runtime data, no timestamps/turn-counters/UUIDs/`datetime.now()`, no `dict.items()`
ordering risk, no JSON-in-prefix whitespace drift. The KB slot is an **empty trailing segment**
(persona → KB → history → turn order frozen NOW). PERS-07's correction-aggressiveness IS the
`CORRECTION` enum; default `gentle` must preserve today's PERS-01 behavior.

**Sandbox-verifiable** (no livekit import needed): `python3 agent/persona.py` runs the
self-check; `python3 -m py_compile` both files. **Operator gate** (VM): the live agent still
greets and converses identically with the default persona. Marked `autonomous: false`.

This plan touches **PERS-07** directly (the `CORRECTION` enum is its mechanism). PERS-02/03/04/05/06
are wired live in 03-02 but their agent-side substrate (knob tables, voice list, render order,
named `agent` ref) is built here.

## must_haves

truths:
- PERS-07: correction-aggressiveness is an enum (`gentle|moderate|aggressive`); the default trainer's gentle correction is the `gentle` value and is `DEFAULT_PERSONA`'s default.
- The rendered persona prefix is byte-stable: same `Persona` → identical bytes, asserted across repeated calls and across knob permutations that share an enum value (`_self_check`).
- The prompt is assembled as frozen constants in a fixed order `role → difficulty → verbosity → correction → footer → KB_SLOT(empty)` — the Phase-4 frozen-prefix seam.
- `render_persona(DEFAULT_PERSONA)` is behaviorally equivalent to today's static `PERSONA_INSTRUCTIONS`; PERS-01 / DEPLOY-03 default-on-load behavior is unchanged.
- `agent/main.py` holds a single NAMED `agent` reference reachable by a future RPC handler.

must_haves.prohibitions:
- No interpolated numbers / f-strings / `.format` on runtime data in the persona prefix; no `dict.items()` ordering dependence.
- No volatile data (timestamps, turn counters, UUIDs, `datetime.now()`, random) anywhere in the rendered prefix.
- No change to `agent/metrics.py` — the per-turn JSON key set (`eou_ms/stt_ms/llm_ttft_ms/tts_ttfb_ms/e2e_ms/over_budget`) and `attach()` stay byte-identical.
- No `AgentSession` re-plumbing (transport, STT/LLM/TTS construction, turn_handling, warmup) beyond sourcing the voice from the persona.
- No second hardcoded LLM tag — the model still resolves from `OLLAMA_MODEL` via `resolved_llm_tag()`.
- Do NOT add the RPC handler or `update_instructions`/`update_options` calls in this plan (that is 03-02).
- KB slot must not be reordered or filled — it stays `""` at the tail.
