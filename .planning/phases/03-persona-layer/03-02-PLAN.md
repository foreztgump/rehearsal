# Plan 03-02: Live-editable persona — side panel → RPC → in-session hot-swap (no restart)

## User Story

**As a** learner, **I want to** edit the persona's role/instructions and display name, adjust the
difficulty/verbosity/correction knobs, and pick a Kokoro voice in a side panel and hit Apply,
**so that** the trainer changes within the current session without a restart — with an
"applying…/applied" signal so I know it took (one-turn re-prefill).

## Context

This is the **UI + control-channel + live hot-swap** slice that completes Phase 3. It builds on
03-01's `agent/persona.py` (knob tables, voice list, `render_persona`, `Persona` dataclass) and
the named `agent` ref already in `entrypoint()`. Three moving parts (`03-PATTERNS.md` Patterns
B2/C/D):

1. **`web/app/PersonaPanel.tsx`** (new client component) — role `<textarea>` (PERS-02),
   display-name `<input>` (PERS-03), three knob `<select>`s (PERS-04), a voice `<select>` from
   the frozen voice id list (PERS-05), an Apply button, and an `"idle"|"applying"|"applied"|"error"`
   status line. Mirror `AgentStatePill.tsx` (thin `"use client"`, inline styles, a
   `@livekit/components-react` hook) and `VoiceRoom.tsx`'s try/catch+error-state form pattern.
2. **RPC control channel** — Apply calls `room.localParticipant.performRpc({ destinationIdentity,
   method: "persona.update", payload: JSON.stringify(persona) })`. The native RPC return value IS
   the "applying…→applied" ack (no custom protocol). Agent registers
   `register_rpc_method("persona.update", handler)` in `entrypoint()` after `ctx.connect()`.
3. **Hot-swap handler** (`agent/main.py`) — parses the snapshot into a `Persona`, `await
   agent.update_instructions(render_persona(p))` (effective next turn), `session.tts.update_options(
   voice=p.voice_id)` (effective next utterance), returns `"applied"`. **No `AgentSession`
   teardown** (PERS-06).

**Critical constraints** (`03-RESEARCH.md` §1/§2/§5, `03-PATTERNS.md` E):
- Use `update_instructions` (async, on the **Agent**) + `update_options` (sync, mutates the
  **existing** `session.tts` so the metrics subscription survives) — NOT session recreate, NOT TTS
  plugin recreate.
- Do NOT change `agent/metrics.py`. The re-prefill turn after an edit shows **elevated
  `llm_ttft_ms`** and correctly flags `over_budget: ["llm_ttft"]` for that one turn — **expected,
  do not "fix"**.
- Full-snapshot, idempotent apply (last-edit-wins) → spam-safe; the in-flight turn finishes under
  the old persona/voice (no mid-word voice flip).
- Client `VOICE_IDS` + default persona seed must **mirror** `agent/persona.py` (file-#6
  duplication — hardcode-to-match; note drift risk, no `persona.get` RPC in MVP).
- The RPC payload shape MUST match `Persona(**snapshot)` field names exactly:
  `{role_text, display_name, difficulty, verbosity, correction, voice_id}`. No secret in the client.

**`[VM-INTROSPECT]` — DEFER, do NOT mark passed** (same discipline as 02-02/02-03). The live
hot-swap, RPC round-trip, and voice-mid-utterance are **operator gates** on the VM. Marked
`autonomous: false`. Sandbox-verifiable: `next build`/typecheck clean, panel renders, RPC method
name + payload shape match, `py_compile` of the handler.

truths:
- PERS-02: the role/instructions textarea edit is carried in the RPC payload and applied via `update_instructions(render_persona(p))`.
- PERS-03: the display-name input is editable in the panel and carried in the payload (UI label; out of the prompt prefix in MVP).
- PERS-04: difficulty/verbosity/correction are three `<select>`s whose enum keys map to fixed-string fragments server-side.
- PERS-05: the Kokoro voice is selectable from the curated list and applied via `session.tts.update_options(voice=...)` on the next utterance.
- PERS-06: persona changes apply within the current session with NO `AgentSession`/`Agent` restart; the RPC return value is the "applying…→applied" ack.
- The persona panel is mounted inside `<LiveKitRoom>` so it shares the room context for `performRpc`.

must_haves.prohibitions:
- No `AgentSession` teardown / `Agent` recreation / TTS-plugin recreation to change persona or voice.
- No change to `agent/metrics.py` (per-turn JSON key set + `attach()` stay byte-identical; the re-prefill turn's elevated `llm_ttft_ms` is NOT "fixed").
- No secret/token in the client payload (persona text/enums/voice id only).
- No per-turn persona editing / auto-edit (would bust the frozen prefix every turn) — edits are user-initiated Apply only.
- No live fetch for the voice list (frozen client constant mirroring `agent/persona.py`; reconcile on VM only).
