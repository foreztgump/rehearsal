# Plan 06-02: Make the critique actually instructive (rubric structure) and stop the agent cutting in on deliberate interview answers — gated on a strong-vs-weak discrimination check

## User Story

**As a** learner answering mock-interview questions slowly and thoughtfully, **I want** the agent to
(a) give me a *structured, discriminating* critique — praising a strong answer and naming the real
gaps in a weak one — followed by a genuinely strong model answer, and (b) wait through my "let me
think…" pauses instead of cutting in mid-thought, **so that** the practice is instructive and feels
like a real interview rather than a rushed quiz.

## Context

This is the **quality/tuning slice** that deepens the working loop shipped in 06-01. It owns the two
hardest, highest-risk parts of Phase 6:

1. **Critique depth (the E4B blocker, STATE.md line 101 / RESEARCH §6 / Pitfall 11).** A 4B-class
   model gives shallow, generic critique and may praise weak answers. The mitigation is **prompt
   structure, not model size**: replace 06-01's basic `CRITIQUE_CONTRACT` with a **rubric-structured**
   critique block — an explicit, frozen qualitative structure (assess technical accuracy,
   completeness, use of precise practitioner terminology, and answer structure) followed by a fixed
   template (brief critique → strong model answer → next single question). Built as frozen constants in
   `interview.py` (the `DISTILL_INSTRUCTION` shape). **The phase is GATED on a strong-vs-weak answer
   discrimination check** — the must_haves truth below — exactly what the STATE.md blocker demands. If
   E4B fails it, the documented 24GB fallback is the trigger.

2. **Slow-speech endpointing re-tune (MODE-05 / RESEARCH §5 / Pitfall 6).** Interview answers are
   pause-heavy; the conversational profile (`min_delay 0.3 / max_delay 3.0` at `agent/main.py:182`)
   cuts in mid-thought. The interview profile raises the floor (`min_delay ~0.6–0.8`, `max_delay
   ~5.0–6.0`) so a thoughtful pause isn't read as turn-end, keeping `MultilingualModel()` as the
   semantic decider.

3. **24GB fallback DOCUMENTED, not built (RESEARCH §6 24GB section).** The LLM is already behind
   LiveKit's interface (`with_ollama(model=resolved_llm_tag())`, tag from `OLLAMA_MODEL`), so swapping
   to a bigger model is a config change. 06-02 ships a doc section (swap mechanism, VRAM math, trigger
   condition) — NO 24GB code path.

**The open design question — how to SWITCH endpointing profiles — is UNRESOLVED and MUST be flagged,
not silently decided (RESEARCH §5 / §9 q1-q2, downstream-consumer requirement).** There is no proven
runtime `turn_handling` setter in this codebase. The plan carries it as an explicit ASSUMPTION and
orders the mechanisms: (1) `[VM-INTROSPECT]` whether `Agent.__init__` accepts `min_endpointing_delay`
/ `max_endpointing_delay` / `turn_detection` (per-agent override — cleanest); (2) `[VM-INTROSPECT]`
whether `session.update_options(...)` can mutate turn options at runtime (memory note suggests NO);
(3) MVP-safe fallback — set the interview-friendly profile as the single session profile OR document
mode-before-start. **Do NOT assume a runtime setter exists.** Because Option B (06-01) keeps a single
agent (no `InterviewAgent` instance), mechanism 1 has no clean carrier unless the VM probe says
otherwise — so the realistic MVP landing is mechanism 3 (a re-tuned single session profile), with
mechanisms 1/2 captured as the VM probe + a documented upgrade path.

**Metrics interpretation (RESEARCH §2.4 / §7.7):** the raised `min_delay` deliberately exceeds
`BUDGET_MS["eou"]=300` (`metrics.py:30-37`), so interview turns flag `over_budget:["eou"]`. **This is
EXPECTED and correct, not a regression** — the verify doc must state this so the metrics line isn't
misread. `agent/metrics.py` is READ-ONLY here.

**Sandbox vs VM split:** the rubric constants + the updated `_self_check` in `interview.py` are
sandbox-verifiable. The endpointing profile edit is sandbox-safe to write (`py_compile`) but its
runtime effect and the strong-vs-weak critique quality gate are **operator/VM gates** (no Ollama/voice
loop in the sandbox). Marked `autonomous: false`.
