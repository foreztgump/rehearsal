# Phase 13 — UI Design Contract

> Visual and interaction contract for the UI/UX overhaul: a dedicated landing/setup screen (configure everything before connecting) and a polished talking screen with smart auto-scroll. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Isolation constraint (load-bearing, from CONTEXT.md + v1.1 scope):** Frontend-only. MUST NOT change the server pipeline (agent/STT/LLM/TTS) or the avatar Path-A lip-sync contract. Voice-only mode stays byte-for-byte identical in behavior. The persona prompt is unchanged. The `layout.tsx` importmap MUST remain emitted in `<head>` before any avatar module; the avatar stays dynamic-imported (`ssr:false`) and absent from the voice-only bundle (AVTR-01). No new styling/animation dependency may ship into the voice-only bundle.

Rationale: (1) isolation constraint forbids bundle-weight regression for voice-only and a Tailwind+Radix migration is a large, cross-cutting change; (2) the stack is bleeding-edge (Next 16.2.9 / React 19.2.7) where shadcn presets are not validated; (3) an internally-consistent dark palette already exists across 6 panels and is the de-facto design system. **Decision: extract the existing inline tokens into one shared `web/app/ui/tokens.ts` (or `theme.ts`) module and refine, rather than introduce a framework.** Registry safety gate: not applicable.

### Token module contract (the refactor seam)

The duplicated `panelStyle` / `labelStyle` / `inputStyle` / `STATUS_LABEL` / `STATUS_COLOR` blocks (copy-pasted across `PersonaPanel`, `ModelPanel`, `KbPanel`, `InterviewPanel`) MUST be consolidated into one exported module consumed by every panel. Constants that mirror the agent (`VOICE_IDS`, `CHOICES`, `ROLES`, `MODE_*`, `DEFAULT_PERSONA`) MUST stay byte-identical to their current values — this refactor is presentational only and must not drop or rename a single agent-mirrored key (the duplication-seam invariant from CONTEXT.md).

## Spacing Scale

Declared values (multiples of 4). Existing rem-based ad-hoc spacing (`0.25/0.4/0.5/0.6/0.75/1rem`) is normalized to this px scale:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status-line padding, inline label-to-control gap |
| sm | 8px | Compact element spacing, control internal padding, segmented-toggle gaps |
| md | 16px | Default gap between fields, panel internal padding, button padding-y baseline |
| lg | 24px | Card/section padding, gaps between setup groups |
| xl | 32px | Layout gaps between major regions (transcript ↔ avatar ↔ controls) |
| 2xl | 48px | Setup-panel outer margin, major section breaks |
| 3xl | 64px | Page-level vertical rhythm (landing hero spacing) |

Exceptions: pill/segmented-control border-radius uses `999px` (full-round, already established); control border-radius `8px`; card border-radius `12px`. Minimum interactive target height `40px` (buttons/inputs/selects) — bump to `44px` for the primary Start CTA touch target.

## Screen & State Contract

This phase introduces a **two-screen** flow; the contract below is the source of truth for the planner/executor.

### Screen A — Landing / Setup (outside `<LiveKitRoom>`, no room yet)

- **Single elegant panel** (D-01), NOT a wizard, NOT a card grid. One organized surface centered on `#0b0f14`, max-width ~`720px`, card background `#0d1117`, radius `12px`, padding `lg`→`2xl`.
- Wordmark "Adept" (Display) + one-line tagline (Body, muted) at top.
- Config groups, each a labeled section with `lg` vertical rhythm: **Persona**, **Knowledge base**, **Response model**, **Interview mode**, **Microphone**, **Avatar (Voice only / Avatar)**. All pre-filled with sensible defaults (D-02): Cybersecurity Trainer persona, Fast model, Learn mode, default mic, Avatar OFF.
- **Progressive disclosure:** advanced/optional fields (persona role text, voice, difficulty/verbosity/correction, interview role) may sit behind an expandable "Customize" affordance so the default path is one glance → Start. The collapsed state shows the current default as a summary line.
- **Primary CTA** pinned/prominent: `Start session` (green `#3fb950`, Display-adjacent weight, `44px` target). It is NEVER disabled by missing choices (D-02) — defaults make it always valid.
- **Microphone selection:** enumerate input devices via `navigator.mediaDevices.enumerateDevices()`. Device labels are only populated after a permission grant — show a "Allow microphone to list devices" affordance; selection is optional (a chosen `deviceId` is passed into LiveKit `audioCaptureDefaults` on connect). This is net-new client state, no server change.
- **Pre-connect config holding (KEY TENSION — CONTEXT.md):** Persona/Model/Interview today apply via RPCs that require a live agent, and KB rides the open room as a byte stream. On the setup screen there is NO room. Contract: **hold every choice in client state on the setup screen; on Start, connect once (single user gesture: mic-permission + autoplay-unlock + connect), then on room-connected apply the held persona/model/interview snapshots via their existing RPCs and fire any queued KB upload over the now-open stream.** The setup screen must surface a brief "Connecting…" → "Applying your setup…" transition so the apply-after-connect work is visible, not a dead pause. No RPC/stream contract changes — only *when* they fire moves.

### Screen B — Talking (inside `<LiveKitRoom>`)

- Layout regions (replace today's flat flex row of 5 panels): a top **bar** (agent-state pill + Voice-only/Avatar toggle + a "Back to setup" / settings affordance), an optional **avatar stage** (only when Avatar ON; `360px` region, unchanged mount contract), the **transcript** as the primary focal column (auto-scroll, see below), and **session controls** (config panels accessible but not crowding the transcript — e.g. a collapsible side drawer or a "Settings" disclosure, so the conversation is the hero).
- Agent-state pill colors unchanged (`AgentStatePill.tsx`): initializing/idle `#8b949e`, listening `#3fb950`, thinking `#d29922`, speaking `#58a6ff`.
- The headphones tip currently on the Start button moves to the setup screen (pre-connect context), not the talking screen.

### Transcript auto-scroll contract (success criterion 4 — load-bearing)

`Transcript.tsx` renders a `<ul>` of `useTranscriptions()` segments. Add a scroll container with **stick-to-bottom** behavior:

- Maintain an `atBottom` ref/state: user is "at bottom" when `scrollTop + clientHeight >= scrollHeight - threshold` (threshold `~32px`).
- On new/updated segments: if `atBottom`, scroll to bottom (newest line in view). If the user has scrolled up (`!atBottom`), DO NOT move the viewport (no yank).
- When `!atBottom` and new content arrives, show a non-intrusive **"Jump to latest ↓"** pill (accent `#58a6ff`, bottom-center of the transcript) that returns to bottom + re-engages stick-on-click.
- Interim (in-progress) segments are styled distinctly from finals (e.g. finals full-opacity, interim `0.7` opacity / italic) — preserves the existing two-sided user/agent split (user right `#e6edf3`, agent left `#58a6ff`).
- Scroll behavior `auto` (instant) for streaming token updates to avoid animation thrash; `smooth` only for the explicit "Jump to latest" click.

### Empty / loading / error states

- **Transcript empty:** heading "Start talking" + body "Say hello, ask a question, or describe what you want to practice." (muted, centered in the transcript column).
- **Connecting:** the Start CTA enters a non-janky busy state ("Connecting…") and the screen transitions to "Applying your setup…" before the talking UI appears; never a frozen dead-button.
- **Token/connect error:** existing `#f85149` message pattern, but actionable: "Couldn't reach the session server. Check the stack is running, then try again." with the CTA returning to its ready label.
- **Mic listing blocked:** inline note (not a hard error) — selection stays optional.

## Animation Contract

Tasteful, performant, no jank (D-03). **CSS-only — no animation library** (bundle-weight isolation). Use CSS transitions/`@keyframes` and `transform`/`opacity` (GPU-friendly), never animate `width`/`height`/`top`/`left` for motion.

| Interaction | Property | Duration | Easing |
|-------------|----------|----------|--------|
| Setup → Talking screen transition | `opacity` + `transform: translateY` (fade/slide) | 240ms | `ease-out` |
| Talking → Setup (reverse) | `opacity` + `transform` | 200ms | `ease-in` |
| Hover/focus on buttons & inputs | `background`, `border-color`, `box-shadow` (focus ring) | 120ms | `ease` |
| Segmented-control selection slide | `transform` of active indicator | 160ms | `ease-out` |
| Status `applying…→applied` color/label | `color`, `opacity` | 150ms | `ease` |
| "Customize" disclosure expand/collapse | `grid-template-rows` 0fr→1fr or `max-height` + `opacity` | 200ms | `ease-out` |
| Agent-state pill change | `background-color` | 150ms | `ease` |
| "Jump to latest" pill in/out | `opacity` + `transform: translateY` | 150ms | `ease-out` |

**Reduced motion:** wrap all transitions in `@media (prefers-reduced-motion: reduce)` to disable transforms/opacity transitions (instant state changes) — required for accessibility (success criterion 5). Transcript token streaming MUST NOT animate (instant render) regardless.

## Navigation Contract (success criterion 3 — reversible, non-breaking)

- "Back to setup" from the talking screen is **reversible without breaking the live room**. Two acceptable patterns (executor's choice): (a) **overlay settings** — open setup as a modal/drawer over the live room so the room connection stays open and changes apply via the same live RPCs (no disconnect); or (b) **explicit leave** — a confirmed disconnect that returns to setup and reconnects on next Start.
- **Recommended: pattern (a)** for in-session tweaks (persona/model/interview/KB all already apply to a live agent), reserving a separate, clearly-labeled **destructive** "Leave session" for tearing down the room. This avoids a disconnect/reconnect for a simple option change and matches the existing live-apply architecture.
- Keyboard: primary actions reachable by Tab; Start CTA and "Jump to latest" activate on Enter/Space; Escape closes the settings overlay/drawer. Visible focus ring (accent `#58a6ff`, `2px`) on every focusable control.
