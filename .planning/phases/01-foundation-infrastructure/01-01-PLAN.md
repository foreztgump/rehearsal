# Plan 01-01: Docker Compose stack + GPU passthrough + HTTPS-on-LAN

## User Story

**As a** homelab operator, **I want to** run one `docker compose up` that boots all six
services with the GPU visible to the model containers and the web shell served over a
LAN-trusted HTTPS cert, **so that** the full stack is reachable and `navigator.mediaDevices`
is defined on a real LAN device — the precondition for every later voice slice.

## Context

GREENFIELD — no source exists yet. Follow reference templates **T1** (Docker GPU Compose),
**T7** (mkcert HTTPS secure context) from `01-PATTERNS.md` verbatim. This plan stands up the
skeleton's outer shell: the compose manifest, GPU reservation syntax, the TLS reverse proxy,
the mkcert CA-trust procedure, and a minimal Next.js shell that proves the secure context.
Models are pinned/validated in 01-02; LiveKit/agent/metrics in 01-03. Images are referenced
here; the web image is custom-built in this plan and a **minimal placeholder `agent/Dockerfile`**
(uv-based) is created so a bare `docker compose up` can build and boot all six services at this
boundary — 01-03 fleshes out the agent image (deps + weights bake) on top of this placeholder.

This is **wave 1** — nothing depends on the others yet; everything depends on this.
