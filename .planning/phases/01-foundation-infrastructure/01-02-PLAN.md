# Plan 01-02: Ollama model pin + flash-attn/KV-quant + VRAM validation under load

## User Story

**As a** latency-sensitive operator, **I want to** pin the exact Gemma 4 quant with thinking
disabled and flash-attn + q8_0 KV cache engaged, then prove all three models co-reside under
the 16GB VRAM floor under concurrent load, **so that** the foundation's defended VRAM budget
and flat-TTFT precondition are real and measured, not assumed.

## Context

GREENFIELD. Follow template **T2** (Ollama env + model config) from `01-PATTERNS.md`. This
plan resolves two open risks flagged in RESEARCH.md and STATE.md:
1. The success-criterion model tag `gemma4:e4b-it-q4_K_M` is **UNVERIFIED** — it may not be a
   real Ollama tag. This plan pull-tests it with the **fallback ladder** and logs the decision.
2. `OLLAMA_KV_CACHE_TYPE=q8_0` silently falls back to F16 if Gemma 4 is not on Ollama's
   flash-attn allowlist (STATE.md blocker line 67) — must be verified empirically.

Depends on 01-01 (compose + GPU passthrough must be working). The warmup script here produces
the **first real LLM TTFT**, which 01-03's metrics scaffold will route as the walking-skeleton
"one real metric" gate. Whisper + Kokoro are already declared in compose from 01-01; this plan
validates them co-resident, no new STT/TTS code yet.
