# Plan 02-01: Browser SPA + LiveKit SDK + agent-state pill + two-sided transcript

## User Story

**As a** learner, **I want to** open one page, click "Start talking," and immediately be
joined into a live audio room where I can see the agent's state (listening/thinking/speaking)
and a running two-sided transcript, **so that** I have a hands-free voice UI wired to the
already-built agent worker with no setup before the first turn.

## Context

This is the first vertical slice: **thin end-to-end "I can join and hear."** Phase 1 already
built the agent `AgentSession` (`agent/main.py`), the token route
(`web/app/api/token/route.ts` — REUSE AS-IS, do not touch), and proved secure-context
mediaDevices over Caddy TLS. The web app is currently a static "stack online" probe; this plan
replaces it with the real room-join SPA using `@livekit/components-react` hooks, not raw
`livekit-client` plumbing.

Follow `02-PATTERNS.md` Pattern A (deps), Pattern B1 (room wrapper + entry button), B3 (state
pill), B4 (transcript), Pattern F (`NEXT_PUBLIC_LIVEKIT_URL`). The net-new components mirror
the shape of the existing `web/app/SecureContextProbe.tsx` (`"use client"`, typed state,
inline-styled UI — the repo uses inline `style` objects, no CSS framework).

Key constraints carried from Phase 1 / AGENTS.md:
- **Pin exact npm versions** — never float a tag. Resolve concrete `livekit-client` and
  `@livekit/components-react` versions at plan-execution time via `npm view <pkg> version`
  (mirror how 01-01 resolved image tags), and commit the regenerated lockfile.
- `livekit-client` major must match the self-hosted server line (server v1.10.x ↔ client 2.x
  per STACK.md).
- Browser connects to `wss://<lan-host>:7443` (Caddy TLS vhost → livekit-server:7880).
- The entry button is a single user gesture that does mic-permission + autoplay-unlock +
  connect — `<StartAudio/>` is the autoplay backstop (TL;DR #6).

**Sandbox limitation (carried):** no Docker daemon, no browser here. Build/type-check/grep are
client-verifiable; the actual room join + audio + pill + transcript are **operator gates** on
the Proxmox VM with a real CA-trusted LAN device (`LIVEKIT_NODE_IP` set, UDP 7882 / TCP 7881
open). Plan marked `autonomous: false` for the human-in-the-loop join verification.
