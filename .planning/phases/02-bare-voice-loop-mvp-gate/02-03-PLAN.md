# Plan 02-03: Tune + measure — barge-in, endpointing, AEC, per-turn v2v latency

## User Story

**As a** learner mid-conversation, **I want to** interrupt the agent instantly by talking over
it, have it wait for me to finish a thought (not cut in on mid-sentence pauses), not hear it
echo itself, and have every turn's voice-to-voice latency measured, **so that** the loop feels
live (P50 < ~1.2s this phase) and the flat-TTFT measurement contract exists for later phases.

## Context

Barge-in and first-sentence streaming are **built into `AgentSession` — this is a TUNE/VERIFY
plan, not a build plan.** The metrics scaffold (`agent/metrics.py`) is wired per-plugin but
only the warmup TTFT has ever fired; Phase 2's real turns now populate it, and this plan adds
the missing end-to-end voice-to-voice number + real rolling P50/P95.

Follow `02-PATTERNS.md` Pattern D (endpointing + barge-in tuning in `agent/main.py`) and
Pattern E (real per-turn + e2e metrics in `agent/metrics.py`). Client-side AEC constraints were
already added in 02-01 (`audioCaptureDefaults`); this plan VERIFIES them under the worst-case
acoustic scenario, it does not re-add server-side plugins.

**BLOCKER — endpointing API surface (do FIRST in this plan).** `livekit-agents~=1.5` exposes
ONE of two mutually-incompatible config surfaces; passing the wrong one throws `TypeError`.
Introspect the installed signature before writing any endpointing config:
`python -c "import inspect, livekit.agents as a; print(inspect.signature(a.AgentSession.__init__))"`.
- Surface 1 (stable 1.5 — direct kwargs, seconds): `AgentSession(..., min_endpointing_delay=0.3, max_endpointing_delay=...)`
- Surface 2 (newer — `TurnHandlingOptions` / `endpointing` dict): `AgentSession(..., turn_handling=TurnHandlingOptions(turn_detection=MultilingualModel(), endpointing={"mode":"dynamic","min_delay":0.3,"max_delay":...}))`
Target `min_endpointing_delay` **~250–350ms** regardless of which surface carries it (default
500ms is half the P50 budget). Prefer `dynamic` mode if available.

Other verify-before-use kwargs (open question 2): `min_interruption_duration`,
`false_interruption_timeout` / `resume_false_interruption`, Silero VAD `activation_threshold`
(set at `silero.VAD.load(...)` in `prewarm()`). Degrade gracefully if a kwarg is absent.

**Local-first (PERF-03):** echo defense is the browser-side AEC from 02-01 — do NOT add
server-side Krisp / `ai_coustics` noise-cancellation plugins. Metrics are **local logs only** —
no Prometheus/Opik/OTEL export.

**Metrics contract (Nyquist):** keep the structured per-turn line format STABLE — Phase 4/5
assert turn-2 TTFT ≪ turn-1 and no creep over N turns against this baseline.

**Sandbox limitation:** no Docker/GPU/browser here. Signature introspection, `py_compile`,
unit-testable percentile math, and source-grep are client-verifiable; live barge-in timing,
slow-speech endpointing, acoustic echo, and the real P50 number are **operator gates** on the
Proxmox VM + real LAN device (speakers+mic worst case). Marked `autonomous: false`.
