## Role / data-flow classification

**Transport-in (mic → agent):** `<LiveKitRoom audio>` publishes the browser mic track →
livekit-server (`wss://...:7443`) → agent worker's `AgentSession` (VAD → MultilingualModel
turn-detect → faster-whisper STT). File #4 owns the browser side; #9 already owns the agent
side (session constructed in Phase 1).

**Compute (turn loop):** #9 `agent/main.py` — `generate_reply` (greeting) + automatic
per-turn replies (LLM via Ollama → sentence-split → Kokoro TTS first-sentence stream).

**Transport-out (agent → speakers):** TTS audio track → livekit-server → browser
`<RoomAudioRenderer/>` (#4), gated by `<StartAudio/>` for the autoplay policy.

**UI-state:** #5 (`useVoiceAssistant().state` pill) + #6 (`useTranscriptions()` transcript),
both read-only views over room state. The agent's `lk.agent.state` participant attribute and
`lk.transcription` text streams are the data sources — no custom data-channel protocol.

**Observability:** #10 `agent/metrics.py` — per-plugin `metrics_collected` populate on real
turns; add the `e2e_ms` (last-user-audio → first-agent-audio) anchor + real P50/P95.

### B1 — Room wrapper + audio (file #4) — VOICE-01/05, DEPLOY-03, TL;DR #6
Single entry button does mic-permission + autoplay-unlock + connect in one gesture.
```tsx
"use client";
import {
  LiveKitRoom, RoomAudioRenderer, StartAudio,
} from "@livekit/components-react";
import { useState } from "react";

const SERVER_URL = process.env.NEXT_PUBLIC_LIVEKIT_URL!; // wss://<lan-host>:7443

export default function VoiceRoom() {
  const [token, setToken] = useState<string | null>(null);
  if (!token) {
    return (
      <button onClick={async () => {
        const res = await fetch("/api/token");      // reuse #7 unchanged
        const { token } = await res.json();
        setToken(token);
      }}>Start talking</button>
    );
  }
  return (
    <LiveKitRoom serverUrl={SERVER_URL} token={token} connect audio video={false}>
      <RoomAudioRenderer />   {/* plays agent TTS track */}
      <StartAudio label="Click to enable audio" />  {/* autoplay-gesture gate */}
      <AgentStatePill />
      <Transcript />
    </LiveKitRoom>
  );
}
```
- `audio` enables open-mic publish (VOICE-05, no PTT). `video={false}` keeps it audio-only.
- The button click is the secure-context gesture (Phase 1 proved `mediaDevices` defined)
  **and** the autoplay unlock; `<StartAudio/>` is the backstop if policy still blocks.
- Basic mic-denial catch is cheap here; full denial UX (REL-01) is Phase 7.

### B2 — AEC / noise-suppression audio constraints (file #4) — 02-03, Pitfalls 4 & 5
**Client-side echo defense is mandatory and local-first** (the reference playback signal
only exists in the browser). Set constraints explicitly via `RoomOptions.audioCaptureDefaults`
so behavior is deterministic across browsers:
```tsx
<LiveKitRoom
  serverUrl={SERVER_URL} token={token} connect audio video={false}
  options={{
    audioCaptureDefaults: {
      echoCancellation: true,
      noiseSuppression: true,
      autoGainControl: true,
    },
  }}
>
```
**Do NOT** add server-side Krisp / `ai_coustics` noise-cancellation plugins — they are
cloud/licensed and violate PERF-03. Recommend headphones in the UI as the clean path.

"use client";

export default function Transcript() {

### C3 — Thinking-OFF in live turns (latency-critical, open question #3)
Phase 1 set `think=false` only at warmup (`THINKING_ENABLED = False` in `_warmup_llm_ttft_ms`).
`build_session()` uses `openai.LLM.with_ollama(...)` — **verify `with_ollama` forwards
`think:false` on `AgentSession`-driven calls**; a `<think>` preamble destroys TTFT and
breaks first-sentence TTS. Note: thinking-OFF in the `adept-gemma` Modelfile is
**request-driven** (`think:false` passed by callers, Modelfile:25-28), NOT template-stripped,
and the `with_ollama` OpenAI-compat `/v1` endpoint has no native `think` field. If it leaks,
either confirm/pass the option explicitly over `/v1` (e.g. `extra_body`) or ADD a template-level
`<think>` strip to the Modelfile and repoint — do not assume the Modelfile already bakes it off.
Acceptance: no `<think>` in live output.

### C4 — Optional latency win: `preemptive_generation`
Add to `build_session()`'s `AgentSession(...)` if the kwarg exists on the pinned `~=1.5`
(open question #2): `AgentSession(..., preemptive_generation=True)`. Starts the LLM before
end-of-turn commits; the framework discards on a false turn. Measure against false
interruptions; pair with C5 endpointing tuning. First-sentence TTS streaming (VOICE-02) is
already free in `AgentSession` — **verify, don't build** (first-audio ts precedes
full-text ts).

## Pattern D — Endpointing + barge-in tuning (file #9) — 02-03

### D1 — Endpointing (VOICE-04) — BLOCKER: verify API surface first
The pinned `~=1.5` exposes **one of two** mutually-incompatible surfaces (passing the wrong
one throws `TypeError`). Introspect before writing config:
```bash
python -c "import inspect, livekit.agents as a; print(inspect.signature(a.AgentSession.__init__))"
```
- **Surface 1 (stable 1.5 — direct kwargs, seconds):**
  ```python
  AgentSession(..., min_endpointing_delay=0.3, max_endpointing_delay=...)
  ```
- **Surface 2 (newer — TurnHandlingOptions / endpointing dict):**
  ```python
  AgentSession(..., turn_handling=TurnHandlingOptions(
      turn_detection=MultilingualModel(),
      endpointing={"mode": "dynamic", "min_delay": 0.3, "max_delay": ...}))
  ```
Target `min_endpointing_delay` **~250–350ms** regardless of surface (default 500ms is half
the P50 budget). Prefer `dynamic` mode if available. Test on hesitant speech ("let me
think… the answer is…") — no cut-in mid-thought, no dead air after a clear finish.

### D2 — Barge-in gate (VOICE-03) — built in, tune the gate
`AgentSession` already cancels TTS + rolls back the turn on user speech. Verify exact kwarg
names on the pinned version (open question #2) before relying on any:
- `allow_interruptions=True` (default)
- `min_interruption_duration` — require N ms of speech before canceling (defends against
  the agent's own echo tail + "mm-hmm" backchannels)
- Silero VAD `activation_threshold` — raise `0.5 → 0.6–0.7` in noisy rooms (Pitfall 4). The
  VAD is loaded in `prewarm()` (`silero.VAD.load()`); pass the threshold there.
- `false_interruption_timeout` / `resume_false_interruption` — if present, enable so a
  noise-blip barge-in (no transcript) resumes instead of dropping the turn (open-mic win).

Degrade gracefully if a kwarg is absent on the pinned line (the research flags each as
verify-before-use).

## Pattern E — Real per-turn + e2e metrics (file #10 `agent/metrics.py`) — 02-03

### E1 — Per-turn fields populate on real turns (verify, no rebuild)
The per-plugin handlers already exist and are wired by `attach()`:
```python
def _on_llm_metrics(metric):  emit_turn(llm_ttft_ms=_seconds_to_ms(getattr(metric,"ttft",None)))
def _on_stt_metrics(metric):  emit_turn(stt_ms=_seconds_to_ms(getattr(metric,"duration",None)))
def _on_tts_metrics(metric):  emit_turn(tts_ttfb_ms=_seconds_to_ms(getattr(metric,"ttfb",None)))
def _on_vad_metrics(metric):  emit_turn(eou_ms=_seconds_to_ms(getattr(metric,"end_of_utterance_delay",None)))
```
Phase 2 drives real turns → these fire (no longer null). **Keep the structured-line format
stable** — it is a validation contract for Phase 4/5 (assert turn-2 TTFT ≪ turn-1, no creep
over N turns — the flat-TTFT Nyquist invariant).

### E2 — Add the e2e voice-to-voice anchor (`e2e_ms`, currently always null)
`emit_turn(...)` already accepts `e2e_ms` and threads it into the record:
```python
record = { ..., "e2e_ms": e2e_ms, "over_budget": [...] }
```
But `e2e_ms` is never passed and is **not recorded into a rolling window** (E3 fixes that).
Define the v2v anchor = last user audio frame → first agent audio frame. Prefer a LiveKit
built-in v2v field on `MetricsReport`/`EOUMetrics` if the pinned version exposes one (open
question #6); otherwise compute from stage timestamps and emit per turn.

### E3 — Implement real rolling P50/P95 (the stub at `metrics.py:63`)
**Analog (current — already real-ish via `quantiles`, but `e2e` has no window):**
```python
def rolling_percentiles(stage: str) -> dict[str, float | None]:
    window = _samples.get(stage)
    if not window or len(window) < 2:
        return {"p50": None, "p95": None}
    cuts = quantiles(window, n=100, method="inclusive")
    return {"p50": round(cuts[49], 1), "p95": round(cuts[94], 1)}
```
`emit_turn()` records only `eou/stt/llm_ttft/tts_ttfb` into `_samples` (note the `stages`
dict excludes `e2e`):
```python
    stages = {"eou": eou_ms, "stt": stt_ms, "llm_ttft": llm_ttft_ms, "tts_ttfb": tts_ttfb_ms}
    for stage, value in stages.items():
        _record(stage, value)
```
**Phase 2:** add `e2e` to the recorded windows so `rolling_percentiles("e2e")` works, and
emit a periodic P50/P95 summary over `ROLLING_WINDOW=100`. Add an `e2e` budget if desired
(gate: **e2e P50 < ~1.2s**). Keep **local logs only** — no Prometheus/Opik/OTEL (PERF-03).
Optionally surface the latest `e2e_ms` in the UI for tuning, but a logged number satisfies
VOICE-08.
