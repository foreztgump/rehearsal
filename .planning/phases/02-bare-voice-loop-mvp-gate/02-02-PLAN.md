# Plan 02-02: Make it converse — greeting + per-turn loop + Cybersecurity Trainer persona

## User Story

**As a** learner who just joined the room, **I want to** be greeted by the Cybersecurity
Trainer and then hold a full spoken back-and-forth (I speak → it answers out loud, starting on
its first sentence), **so that** the bare voice loop actually talks — the hard MVP gate's core.

## Context

The `AgentSession` is fully constructed in `agent/main.py` (VAD + faster-whisper STT + Ollama
LLM + Kokoro TTS + local `MultilingualModel`) and `session.start(...)` is already called — but
**nothing speaks** because there is no greeting and no reply is ever generated, and the
persona is a generic placeholder. This plan turns the loop on. It is **MODIFY against a live
analog**, not greenfield.

Follow `02-PATTERNS.md` Pattern C: C1 (greeting via `generate_reply` after `session.start`),
C2 (Cybersecurity Trainer persona replacing the placeholder at `main.py:53`), C3 (verify
thinking-OFF holds in live turns), C4 (optional `preemptive_generation`). First-sentence TTS
streaming (VOICE-02) is **built into `AgentSession` — verify, do not build** a sentence
splitter.

Critical points from RESEARCH.md:
- Per-turn replies need **no manual glue** — with the full pipeline wired, `AgentSession` runs
  a turn automatically when the user finishes speaking. Confirm default behavior.
- Greeting trigger (open question 8): `generate_reply` after `session.start` vs. on first
  participant connect — pick the form that reliably fires exactly once when the browser joins.
- Thinking-OFF (open question 3): Phase 1 set `think=false` only at warmup; verify
  `with_ollama`-driven `AgentSession` LLM calls do not emit a `<think>` preamble (it would
  destroy TTFT and break first-sentence TTS). Note `with_ollama` connects over the
  OpenAI-compat `/v1` endpoint, which has **no native `think` field** (that lives on Ollama's
  `/api/generate`); and the current `ollama/Modelfile` (lines 25-28) makes thinking-OFF
  **request-driven** (`think=false` passed by callers), NOT template-stripped — so `adept-gemma`
  carries **no baked thinking-off**. If a `<think>` preamble can leak, the fix is to confirm how
  `think:false` is plumbed over the OpenAI-compat path (explicit option / `extra_body`), or to
  ADD a template-level `<|think|>` strip to the `adept-gemma` Modelfile and point the LLM at it —
  do not assume the current Modelfile already strips thinking.
- **Forward-compat (Phase 3 frozen prefix `[persona]+[KB]+[history]+[turn]`):** write the
  persona as the static top block; put NO volatile data in the system prompt so Phase 3 can
  slot a KB beneath it without a rewrite. Keep it plain `Agent(instructions=...)`. Do NOT build
  the frozen-prefix machinery now.
- `preemptive_generation` / other kwargs (open question 2): verify the kwarg exists on the
  installed `livekit-agents~=1.5` before passing it; degrade gracefully if absent.

**Sandbox limitation:** no Docker daemon / GPU / browser here. `python -m py_compile` and
source-grep are client-verifiable; the audible greeting + per-turn conversation + first-sentence
streaming are **operator gates** on the Proxmox VM with the stack up and a real LAN device.
Marked `autonomous: false`.

must_haves.prohibitions:

- No second hardcoded LLM tag — the model still resolves from `OLLAMA_MODEL` via `resolved_llm_tag()`.
- No volatile/runtime data interpolated into the system prompt.
- No re-plumbing of the session transport, STT/LLM/TTS construction, or turn detector built in Phase 1.
