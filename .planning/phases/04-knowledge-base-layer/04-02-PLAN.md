# Plan 04-02: Distill once → inject once into the frozen prefix → KB-grounded trainer (no per-turn cost)

## User Story

**As a** learner who uploaded material, **I want to** have it distilled into a compact brief that
the trainer actually references in conversation, **so that** the agent coaches me on my *own*
content — paid for once at session start, never re-charged per turn.

## Context

This is the **distill + inject + compose** vertical slice that completes the KB behavior (04-03 is
the operator-only flat-TTFT/VRAM proof). It builds on 04-01's parser + in-memory `_SessionKb` +
`kb.state` channel. Three moving parts (`04-PATTERNS.md` Patterns B, C, D2/D3, E):

1. **`agent/kb/distill.py`** (new) — a **pure** `build_distill_prompt(text) -> str` (testable in
   the sandbox) plus `distill(text) -> str` (one off-hot-path Ollama call → a compact prose brief +
   a verbatim `FACTS:` list). Reuses the resident model via the `_warmup_llm_ttft_ms` httpx-stream
   style; `think=false`. Latency is invisible (NOT the voice loop), so a bigger `num_predict` is
   fine.
2. **`render_prompt(persona, kb_brief="")`** in `agent/persona.py` — fills the existing **empty
   `KB_SLOT` seam** with the session-frozen brief. `render_prompt(p, "")` MUST be byte-identical to
   today's `render_persona(p)` so the golden regression stays green.
3. **Ingest → distill → inject ONCE** in `agent/main.py` — extend `_ingest_kb` to distill the
   concatenated docs, store the brief on `_SessionKb`, and call
   `agent.update_instructions(render_prompt(current_persona, brief))` exactly **once** (the single
   sanctioned re-prefill, mirroring the persona hot-swap). Then fire a **priming turn** to warm the
   KB prefill while the panel shows "ready", and **compose with persona edits** so editing the
   persona re-emits the current brief (never clobbers the KB).

**THE keystone constraint is byte-stability of the frozen prefix** (`04-PATTERNS.md` Pitfall 7).
The brief is injected **ONCE per session, then FROZEN**: no re-distill / re-inject per turn; the
brief string must carry **no volatile data** (no timestamps, counters, re-serialized JSON whose
key order/whitespace can drift). The "(persona × KB) epoch" model: a persona edit and a KB load are
BOTH one-time, user-initiated re-prefills — they must **compose, not clobber**
(`04-PATTERNS.md` "epoch" section).

**Ollama bug #15260** (`ollama/Modelfile:30`): `think=false` + `format=json` silently drops the
JSON constraint for gemma4. **Do NOT rely on `format=json`** — parse a delimited/markdown
structure (the `FACTS:` delimiter) out of plain text. JSON mode is `[VM-INTROSPECT]` only-if-wanted.

**Sandbox-verifiable:** `render_prompt(DEFAULT_PERSONA, "") == EXPECTED_DEFAULT` (golden unchanged);
`render_prompt(p, FIXED)` deterministic with the brief at the slot position; `build_distill_prompt`
pure-testable; `py_compile`. **Operator gates** (`[VM-INTROSPECT]`, defer): the live Ollama distill
quality, KB grounding (KB-04), the priming turn, and the inject round-trip. Marked
`autonomous: false`.
