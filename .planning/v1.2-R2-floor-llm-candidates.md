# Floor-tier LLM candidates (v1.2 R2)

The catalogue of small LLMs evaluated for the ~6GB **Floor** tier, why each was kept / flagged /
dropped, and what shipped. **What shipped:** `adept-floor` = `hf.co/mradermacher/Huihui-Qwen3-4B-
Instruct-2507-abliterated-GGUF:Q4_K_M` with the correct Qwen3-2507 chat template grafted via
`ollama/Modelfile.floor` (the raw GGUF's bundled Ollama template was broken). `verify-build` GREEN,
~3.45 GB resident on the RTX 5090.

## Selection criteria
- **Budget:** floor LLM ~1.9–3 GB at q4 (after Kokoro ~1 GB; CPU-ONNX STT = 0 GPU on the floor).
- **D8 — abliterated preferred:** the persona must be the SOLE content guardrail across all tiers, so
  the floor should be abliterated/uncensored. Clean (safety-tuned) models re-introduce mid-character
  refusals on offensive-security role-play.
- **Thinking-OFF (hard rule):** prefer **non-thinking-by-construction** (the `*-Instruct-2507` line)
  to eliminate `<think>` leak risk; hybrid models need `/no_think` + must pass `verify-build` Check B.
- **Gate:** every resolved tag passes `ollama/verify-build.sh` (Check A behavioral chat-template
  probe + Check B thinking-off artifact scan) before it's trusted.
- **Trust:** prefer dual-provenance / reputable quantizers (huihui-ai, mradermacher, mlabonne,
  bartowski) over single-uploader community tags.

## D. The template-graft recovery (reusable trick)

A GGUF can pull fine yet ship a **broken Ollama chat template** (multi-turn rendering collapses;
`verify-build` Check A fails with role-tag echo). Recover without dropping the model: pull the
**official base model** (same family) for its correct template, then `ollama create` the abliterated
GGUF with that template grafted. See `ollama/Modelfile.floor` (the Qwen3-2507 template lifted from
`qwen3:4b-instruct`) and the `pull-and-pin.sh` auto-graft step. **Beware:** don't reuse a *hybrid*
Qwen3 template (it force-primes `<think>`) on a *non-thinking* 2507 model — use the matching base.
